Introduction & Motivation

Boost.DI is C++ 03 / C++ 11 header only library providing compile time, macro free constructor dependency injection functionality.

Dependency Injection?

"Don't call us, we'll call you", Hollywood principle

Dependency injection is a software design pattern that implements inversion of control and allows a program design to follow the dependency inversion principle.

testable code, separation between business logic and object creation

Form of dependency injection: constructor injection setter injection member injection interface injection

Service locator - anti pattern in the end factory

Dependency Injection Library?
  • to avoid writing boilerplate, tedious code
  • to write testable, maintainable code
Motivation example 

#include <boost/di.hpp>

class app {
public:
    app(const std::shared_ptr<int>&
      , boost::shared_ptr<int>
      , std::unique_ptr<int>
      , double
      , int
      , const std::string&)
    { }

    void start() { }
};

int main() {
    auto injector = boost::di::make_injector();

    auto my_app = injector.create<app>();

    my_app.start();
    //...

    return 0;
}

include di headers

Define class as usual

make injector

create app

use my_app
